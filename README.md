# Salon-booking-system (名家理髮廳)

* **地圖連結：** https://share.google/VwdpyG9pYoZ6DaOg6

## 商家痛點

* **人力成本高昂**：仰賴電話或手動回覆訊息，剪髮過程中斷，影響服務品質與效率。
* **人工紀錄失誤**：紙本行事曆易造成重疊預約或漏記，導致現場顧客時間衝突。
* **時段計算複雜**：多項服務組合（如：剪+染+燙）的總耗時難以人工即時精算，造成行程空窗或過度緊湊。
* **預約失約率高**：缺乏自動提醒機制，客戶忘記時間或臨時取消，導致店家當日營收損失。
* **資訊不透明**：客戶無法即時查閱剩餘空檔，需反覆來回詢問，降低預約意願。

---

## 解決方案規劃

### 1. 全天候自動化預約

* **LINE LIFF 整合**：將預約功能內嵌於 LINE，客戶無需下載 App 即可 24/7 自助下單，系統自動同步資料庫，取代人工接單。

### 2. 精準排程演算法

* **5 分鐘顆粒度運算**：後端以 5 分鐘為最小單位進行時間軸碰撞偵測，精確計算「多服務組合」後的可用區塊，前端則維持 30 分鐘間隔的簡潔介面。
* **動態時段過濾**：系統自動加總選取服務的總時長，即時過濾掉長度不足的空檔，確保排程不重疊。

### 3. 預約規範自動化

* **24h 門檻邏輯**：系統強制執行「提前 24 小時預約/取消」規則。不滿 24 小時之取消行為由 API 直接攔截並提示，保障店家排程穩定。

### 4. 主動式通知系統

* **Messaging API 串接**：預約成功、變更或取消時，系統自動推播 Flex Message 圖文訊息，提供即時憑證並降低溝通成本。

### 5. 數位化後台管理

* **雲端整合介面**：老闆可透過管理後台即時掌握每日預約名單、服務細節與客戶聯絡資訊，並支援手動錄入電話預約客。

## 系統概述

整合 LINE LIFF 與 FastAPI 的美髮自動化預約系統。

## 使用技術

* 前端：LINE LIFF (GitHub Pages)
* 後端：Python FastAPI (Render / Zeabur)
* 資料庫：MySQL (Aiven)
* 通訊：LINE Messaging API

## 核心邏輯

* 運算單位：底層 5 分鐘，前端顯示 30 分鐘間隔。
* 預約規則：需提前 24 小時預約或取消。
* 服務機制：支援多選服務，自動加總時長並過濾可用時段。

## 資料庫結構

* **users**: LINE ID 與客戶資料。
* **designers**: 設計師清單。
* **services**: 服務項目與耗時。
* **appointments**: 預約主檔 (時段、狀態)。
* **appointment_services**: 預約與服務關聯表。

## 主要 API

* `GET /api/services`: 取得服務選單。
* `GET /api/availability`: 查詢特定日期可用時段。
* `POST /api/appointments`: 建立預約並發送 LINE 通知。
* `DELETE /api/appointments/{id}`: 取消預約 (含 24h 限制邏輯)。
